Executive Summary
The 30-second version for decision-makers

The Sales Playbook v2.0 defines a proven process — from first DM to signed deal — but it currently lives as a static document. Meanwhile, our CRM already has a strong foundation: a unified pipeline, client tracker, KOL database, and Telegram integration.

The opportunity: By embedding the playbook directly into the CRM, we turn a reference document into an executable system. Every stage, script, template, and follow-up cadence becomes a trackable, automated, and measurable workflow.

Core Thesis The playbook says "Phil sends DMs, books calls, Jdot runs discovery, follows BAMFAM." The CRM should enforce this rhythm automatically — surfacing the right action, for the right lead, at the right time — so the team executes instead of remembering.

Gap Analysis
The six modules needed to close the gap
⚑
1. Lead Scoring & Bucket System

Classify every opportunity as Bucket A/B/C with auto-scoring based on signals (budget mentioned, timeline given, decision-maker present). Drive all downstream automation from the bucket.

📅
2. Activity & Meeting Tracker

Log calls, meetings, and messages tied to opportunities. Track BAMFAM compliance (did the call end with a booked next step?). Surface overdue follow-ups.

⚡
3. Automated Sequences

Bucket-driven follow-up cadences. When a lead enters Bucket B, the 21-day nurture sequence auto-queues. When a proposal is sent, the Day 3/7/14 follow-ups queue automatically.

📈
4. Sales Dashboard

Real-time metrics: Phil's DMs sent, response rate, meetings booked. Jdot's close rate, avg deal size, pipeline value. Weekly rhythm scorecards.

📄
5. Proposal & Document Builder

Generate proposals from opportunity data. Auto-populate situation recap, scope, pricing. Track proposal status (sent, viewed, signed). Step-down to Korea Market Audit.

💡
6. Trigger Event Monitor

Track competitor movements, exchange listings, funding rounds, and team hires. Auto-flag relevant Bucket B/C leads for re-engagement when triggers fire.

Proposed Features in Detail
What each module looks like inside the CRM

4A. Lead Bucket System

Every opportunity gets a bucket field and a temperature score (0-100). The bucket determines the playbook track.

Current Pipeline Stages
  • new → contacted → qualified → unqualified
  • deal_qualified → proposal → negotiation → contract → closed_won/lost
  • No temperature or readiness indicator
  • Manual stage progression only
Proposed: Stages + Buckets
  • Same stages, PLUS a bucket field (A/B/C) on every opportunity
  • Auto-scoring based on: budget signal, timeline given, decision-maker identified, Korea mentioned unprompted
  • Bucket drives which sequence fires and which templates surface
  • Bucket can change as new signals arrive

4B. Activity & Meeting Tracker

Every opportunity gets an activity timeline. Calls, messages, proposals, and meetings are logged with metadata. BAMFAM compliance is tracked automatically.

  • NEW Activity log per opportunity: Call happened, message sent, proposal delivered, meeting booked — each with timestamp, owner, and notes
  • NEW BAMFAM tracker: Every call/meeting must have a "Next Step" field. If empty, the card shows a warning badge.
  • NEW Meeting scheduler: Book next meetings inline. Calendar view of upcoming sales meetings across all opportunities.
  • NEW Pre-call prep checklist: Structured form that auto-fills from opportunity data (project basics, token status, competitors, prepared insight drops)
  • ✓ Telegram integration (exists): Link TG messages to the activity timeline automatically via existing webhook

How Sequences Work Technically A new crm_sequences table stores queued actions per opportunity. Weekly Rhythm Report (Auto-generated every Monday)

The playbook defines a weekly rhythm (Section 16). Every Monday morning, the CRM auto-generates a report sent to the team TG channel:

  • Last week: Deals closed, revenue, conversion rates, DMs sent, meetings run
  • This week: Scheduled calls, proposals due, nurture touches queued, follow-ups overdue
  • Alerts: Stale leads (no contact in 7+ days), BAMFAM violations, Bucket B leads ready for re-assessment
08
AI Integration
Making Holo GPT the playbook's co-pilot

We already have Holo GPT with agent orchestration. Here's how we level it up with playbook knowledge:

🤖
Pre-Call Research Agent

Before a discovery call, Holo GPT auto-generates the pre-call prep doc: project basics (from CRM data), token data (from external APIs), competitor Korea presence, 2-3 suggested insight drops, and recommended bucket prediction.

💬
Message Composer

Given the opportunity context and playbook stage, AI drafts personalized messages using the playbook templates as a base — filling in project name, competitor intel, market data. Phil reviews and sends.

📈
Bucket Scoring Engine

After call notes are logged, AI analyzes the notes against playbook criteria (budget mentioned? timeline given? decision-maker present?) and suggests a bucket classification with reasoning.

🔎
Objection Coach

When Jdot logs an objection type on a call, Holo GPT surfaces the playbook's specific response script plus relevant NEPQ framework steps — available in real-time during the call if needed.

💡
Trigger Event Scanner

AI monitors competitor announcements, exchange listings, and funding rounds (via RSS/APIs). When a trigger matches a Bucket B/C lead's category, it auto-creates a re-engagement task with a drafted message.

📄
Proposal Generator

AI drafts a proposal from opportunity data: situation recap, Korea opportunity sizing, scope of work, timeline, pricing. Jdot reviews, customizes, and sends — cutting proposal prep from 2 hours to 20 minutes.

Leveraging What Exists The agentOrchestrator.ts, agentTools.ts, and advancedAIService.ts already provide the agent framework. We add new tools: getOpportunityContext, draftPlaybookMessage, scoreBucket, generateProposal, scanTriggers. The chat interface (/chat) becomes the playbook co-pilot.

Implementation Roadmap
Three phases, incremental value at each step
Phase 1 — Foundation
Pipeline Alignment + Activity Tracking
  • Add bucket field + temperature score to opportunities
  • Add new pipeline stages (DM Sent, Response, Discovery Booked/Done, Proposal Sent/Call)
  • Build crm_activities table and activity timeline UI on opportunity detail
  • Add BAMFAM next_meeting tracking with warning badges
  • Add decision-maker flag to contacts
  • Link message templates to stages + buckets
  • Build pre-call prep checklist form

Value delivered: Every opportunity has a clear bucket, activity history, and BAMFAM compliance. Templates surface at the right stage. Pre-call prep is structured.

Phase 2 — Automation
Sequences + Dashboard + Weekly Reports
  • Build sequence engine (crm_sequences + crm_sequence_steps)
  • Implement all 4 sequences: Fast Track, 21-Day Nurture, Post-Proposal, Monthly Nurture
  • Build Sales Dashboard page with Phil/Jdot metrics and pipeline summary
  • Build sales_metrics_daily table with auto-snapshot Edge Function
  • Auto-generate weekly Monday report to TG channel
  • Build proposal tracker (crm_proposals table + UI)

Value delivered: Follow-ups happen automatically. No lead falls through the cracks. Metrics are real-time. Monday standups have data.

Phase 3 — Intelligence
AI Co-pilot + Trigger Monitoring + Proposal Generator
  • Add playbook knowledge to Holo GPT (new agent tools)
  • Auto-generate pre-call research docs via AI
  • AI-assisted bucket scoring from call notes
  • Build trigger event scanner (RSS/API monitoring)
  • Auto-match triggers to Bucket B/C leads and create re-engagement tasks
  • AI proposal generator from opportunity context
  • Objection handling coach (real-time script surfacing)

Value delivered: The CRM doesn't just track — it thinks. Prep time drops 80%. Proposals are drafted in minutes. No market signal is missed.

Summary: Before & After

Before (Playbook as Document)
  • Team reads 17-section PDF and tries to remember
  • Follow-ups tracked in heads or scattered notes
  • No visibility into DM volume, response rates, or meeting counts
  • Proposals built from scratch each time
  • Bucket classification is a mental exercise
  • Trigger events noticed by chance
  • BAMFAM is aspirational, not enforced
  • Weekly rhythm depends on discipline
  • No way to measure playbook compliance
After (Playbook in CRM)
  • Playbook actions are the CRM workflow — no memorization needed
  • Every follow-up is queued, templated, and tracked
  • Real-time dashboard with all playbook KPIs
  • AI generates proposal drafts from opportunity data
  • Auto-scoring classifies buckets from call data
  • Trigger events auto-scanned and matched to leads
  • BAMFAM enforced with warnings on every opportunity
  • Monday report auto-generated and sent to TG
  • Full audit trail of playbook execution per deal
The Goal When a new salesperson joins, they don't read the playbook — they open the CRM, and the playbook tells them exactly what to do next. Every lead, every day, every touchpoint. No guesswork.
