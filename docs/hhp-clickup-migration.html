Dashboards

Three Dashboard Types

Dashboards are a core structural component — not an afterthought. Each serves a distinct audience and purpose.

My Dashboard

Replaces ClickUp "My Work" view.
Visible to: team member + Partners/Admin

  • All assigned tasks
  • Tasks by status (To Do / In Progress / Review / Done)
  • Overdue tasks highlighted
  • Recurring tasks due this week
  • Upcoming deadlines (7-day view)
Admin Overview

Partners & C-Level only.
Replaces ClickUp team dashboards.

  • Open tasks per team member
  • Overdue count per member
  • Completed this week per member
  • Active tasks per client
  • Overdue per client
  • Internal Ops snapshot
Client View

Per-client internal execution view.
Connected to current campaign manager.

  • Active campaigns
  • Open tasks
  • Upcoming deliverables
  • Overdue items
  • WDR / Delivery Rundowns

Attachments File uploads with preview. PDFs, images, docs.
Comments Threaded discussion with @mentions.
Recurring Auto-recreates task on schedule.

Improvement Over ClickUp

Accountability Enhancements

Fixes the ClickUp problem of "tasks completed but not marked done." These are behavior-correcting systems, not just features.

Overdue TG Bot Alert

Auto-notification via Telegram bot when any task passes its due date. Immediate visibility — no need to open the app.

Weekly Incomplete Task Report

Automated report sent to each team member showing overdue and stale tasks. Admin dashboard mirrors this data.

7-Day Stale Rule

No task can sit "In Progress" for more than 7 days without a comment or status update. Flagged automatically on dashboards.

Friday Auto-Summary

"Tasks completed this week per team member" — sent every Friday. Creates positive accountability and transparency.

Rollout

Migration Phases

A controlled, phased migration that never puts active campaigns at risk. Parallel run before any hard cutover.

0
Feature Parity Audit Week 1
Before moving anything, confirm HHP can support every non-negotiable:
  • Recurring tasks & automation triggers
  • Auto-assign & reminders
  • Form → task creation
  • Full status pipeline replication
If any item fails → migration pauses until resolved. No workarounds.
1
Parallel Run Weeks 2 – 4
Do NOT hard switch. Run both systems simultaneously.
  • New tasks created in HHP only
  • Old tasks continue in ClickUp (no new additions)
  • Monitor: friction, missing features, behavioral gaps
Rule: HHP = new work • ClickUp = legacy work only
2
Active Task Migration Week 5
Only migrate what's alive:
  • Active client tasks
  • Recurring weekly tasks
  • Open sales tasks
Do NOT migrate: completed tasks, archived history, closed campaigns. Historical access stays in ClickUp (read-only).
3
Automation Replication Week 6
Rebuild all automation behavior in HHP:
  • Status change triggers
  • Reminder logic & auto-assign rules
  • Recurring weekly updates
  • Form → task logic
Automations should be behavior-correcting, not just duplicating ClickUp logic. Build in the accountability improvements from Step 6.
4
Hard Cutover Week 7+
Only proceed when all gates pass:
  • All active tasks exist in HHP
  • Recurring tasks confirmed working
  • Automations verified end-to-end
  • All three dashboards stable
Lock ClickUp (no editing) • Announce HHP as sole operational system • Remove ClickUp from daily workflow
