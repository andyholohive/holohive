02
Two-Path Booking Model
The routing key for the entire pipeline

The SDR (Phil) DMs from the Closer's (Jdot's) account and his own. Which account determines the entire downstream flow. A new dm_account field on every opportunity is the routing key — pipeline logic, sequences, and templates all branch on this value.

Path A — Closer's Account

  • Cold DM → Warm → Calendly sent in DM → Booked
  • No TG exchange needed
  • No GC creation needed
  • Lead thinks they're talking to the Closer
  • Faster path — fewer steps to booking

Path B — SDR's Account

  • Cold DM → Warm → Get TG → Calendly in TG → Booked → Open GC
  • TG handoff required before Calendly
  • GC opened after booking (add Closer)
  • Lead doesn't know the Closer yet
  • TG stage has own nudge cadence (Day 2, 5, 10, 14)
Currently Missing The CRM has no concept of which account the SDR is DMing from. No branching logic, no path-specific templates, no Path A vs B metrics. This is the single biggest structural gap.

04
Lead Bucket System
A/B/C classification drives all downstream automation

Every opportunity gets a bucket field and a temperature score (0-100). The bucket determines which sequence fires, which templates surface, and which dashboard metrics apply.

A
Bucket A — Hot

Budget mentioned, timeline given, decision-maker present. Fast-track sequence fires. Target close: 7-14 days.

B
Bucket B — Warm

Interest shown but missing signals. 21-day nurture sequence fires. Re-assess at Day 21 — upgrade to A or move to monthly.

C
Bucket C — Cold

Early stage, no signals yet. Monthly nurture cadence. Monitor for trigger events. Breakup after 4-5 months of silence.

Auto-scoring signals: Budget signal (+25), timeline given (+20), decision-maker identified (+20), Korea mentioned unprompted (+15), responded to DM (+10), booked call (+10). Bucket can change as new signals arrive.

07
Activity Tracking & BAMFAM
Every interaction logged, next steps enforced

Current: Only last_contacted_at and stage history. No per-opportunity activity log. No BAMFAM enforcement.

📝
Activity Log

Every call, message, meeting, proposal logged per opportunity. Timestamp, owner, notes, and outcome tracked.

✅
BAMFAM Enforcement

Every call/meeting must have a "Next Step" field. If empty, opportunity card shows warning badge. next_meeting_at + next_meeting_type fields.

📋
Pre-Call Prep

Structured form auto-filled from opportunity data: project basics, token status, competitors, insight drops, recommended bucket prediction.

💬
TG Integration

Existing Telegram integration linked to activity timeline automatically. Messages show in opportunity context.

Auto-generated and sent to team TG channel every Monday:

  • Last week: Deals closed, revenue, conversion rates, DMs sent (by account), meetings run
  • This week: Scheduled calls, proposals due, nurture touches queued, follow-ups overdue
  • Alerts: Stale leads, BAMFAM violations, no-shows pending, unsigned contracts, Bucket B leads ready for re-assessment

Implementation Roadmap
Four phases, incremental value at each step
Phase 1a — Correct Pipeline + Core Fields
Two-Path Routing + Stages + Cold Sequence
  • Add all Phase 1a fields to crm_opportunities
  • Create crm_activities, crm_sequences, crm_sequence_steps tables
  • Implement correct pipeline stages (Cold DM → Warm → TG → Booked → Discovery Done → Proposal Sent/Call → Contract → Closed Won)
  • Add Orbit + Closed Lost exit lanes with reason fields
  • Add bucket field + temperature score
  • Build cold outreach 5-bump sequence (cron Edge Function)
  • Build activity timeline UI per opportunity
  • Add BAMFAM tracking with warning badges
  • Add decision-maker flag to contacts
  • Implement dedup check on lead creation
  • Update pipeline Kanban/table views for new stages

Value: Pipeline reflects two-path reality. Cold outreach automated. Duplicates prevented. Buckets, activity history, BAMFAM compliance on every opportunity.

Phase 1b — Calendly + Orbit + Templates
Complete the Foundation
  • Calendly webhook (invitee.created → auto-move to Booked)
  • TG booking nudge sequence (Path B: Day 2, 5, 10, 14 → Orbit)
  • Full Orbit system: 5 reasons, two cadences, resurrection flow
  • No-show tracking + 3-day reschedule auto-task
  • Contract alerts (3-day unsigned → alert)
  • Link templates to stages + buckets + dm_account
  • Build pre-call prep checklist form
  • Dashboard: split metrics by dm_account

Value: Calendly auto-updates pipeline. Orbit nurtures long-term. Right template at right stage for right path. No-shows and stale contracts flagged.

Phase 2 — Automation + Dashboard
Sequences + Metrics + Reports
  • Build generic sequence engine for all 7 types
  • Implement: Fast Track, 21-Day Nurture, Post-Proposal Silence, Monthly Nurture, Orbit touches
  • Build Sales Dashboard page (Phil/Jdot metrics, pipeline summary, Path A vs B filter)
  • Build sales_metrics_daily + auto-snapshot Edge Function
  • Auto-generate weekly Monday report → TG channel
  • Build proposal tracker (crm_proposals + UI)

Value: All follow-ups automated. No lead falls through cracks. Real-time metrics with Path A/B comparison. Monday standups have data.

Phase 3 — Intelligence
AI Co-pilot + Triggers + Proposals
  • Add playbook knowledge to Holo GPT (new agent tools)
  • Auto-generate pre-call research docs via AI
  • AI-assisted bucket scoring from call notes
  • Build trigger event scanner (RSS/API monitoring)
  • Auto-match triggers to B/C leads, create re-engagement tasks
  • AI proposal generator from opportunity context
  • Objection handling coach (real-time script surfacing)

Value: CRM doesn't just track — it thinks. Prep drops 80%. Proposals in minutes. No market signal missed.
