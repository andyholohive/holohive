CRM Integration Plan — Audit

Feb 2026 · Reviewed against Sales Playbook v2.0 + Dev Walkthrough + Process Updates

Key context the dev is missing: The SDR [Phil] now DMs from the Closer's [Jdot's] account in addition to his own. This creates two booking paths — the dev's plan predates this update and doesn't account for it. That's the root cause of most issues below.
Critical Fixes — Must Address Before Building 4

1. Two-Path Booking Flow Is Missing

The plan jumps from "Response" to "Discovery Booked" with no concept of how booking actually works. Where the Calendly link gets sent depends on which account the SDR uses.

Path A — Closer's [Jdot's] Account:
Cold DM → Warm → Calendly sent in DM → Booked
No TG. No GC. Lead thinks they're talking to the Closer.

Path B — SDR's [Phil's] / Other Account:
Cold DM → Warm → Get TG → Calendly sent in TG → Booked → Open GC (add Closer)
Lead doesn't know the Closer yet, so TG + GC handoff are required.
What the CRM needs
  • dm_account — which account sent the first DM. This is the routing key for the whole pipeline.
  • calendly_sent_via (dm / tg / not_yet), calendly_sent_date, calendly_booked_date
  • tg_handle, gc_opened (na / not_yet / opened), gc_opened_date
  • Pipeline logic branches on dm_account: Path A skips TG + GC. Path B requires them.

2. Pipeline Stages Don't Match the Process

The plan proposes: DM Sent → Response → Discovery Booked → Discovery Done → Proposal → Contract → Won. Several stages are wrong or missing.

Correct pipeline:
Cold DM → Warm (Interested / Silent) → TG* → Booked → Discovery Done → Nurture** → Proposal Sent → Proposal Call → Contract → Closed Won

*TG skipped for Path A   **Nurture for Bucket B only
Exit lanes at every stage: Orbit (with reason) or Closed Lost
Key fixes
  • "DM Sent" → Cold DM (it's a stage, not an action — leads sit here through 5 bumps)
  • "Response" → Warm with sub-states: Interested vs Silent (Silent triggers re-bump)
  • Add TG stage (Path B only), Orbit (with reason), Closed Lost (with reason)
  • Add warm_sub_state, orbit_reason, closed_lost_reason fields

3. 5-Bump Cold Outreach Sequence Is Missing

The plan treats Cold as if one DM goes out. In reality there's a 5-bump sequence — the highest-volume automation in the system. The plan's 4 sequences all start after discovery. Nothing covers pre-discovery outreach.

Bump 0: Opener → 3-4d → Bump 1: Value → 4d → Bump 2: Credibility → 5d → Bump 3: Direct Ask → 6d → Bump 4: Breakup → 7d → Orbit
What the CRM needs
  • bump_number (0-4), last_bump_date on crm_opportunities
  • New sequence type: cold_outreach — 5 steps with timing rules
  • Bump 4 + 7 days silence → auto-move to Orbit (reason: no_response)
  • Warm-Silent → reset bump to 0, restart the sequence

4. Calendly Isn't Mentioned Anywhere

The word "Calendly" doesn't appear once. But it's the mechanism by which leads go from interested to booked. The plan has "Discovery Booked" as a stage but nothing about what triggers that transition.

What the CRM needs
  • Calendly tracking fields (listed in #1 above)
  • Calendly webhook (invitee.created) → auto-move lead to Booked (Phase 2 if needed)
  • Dashboard: leads where Calendly sent but not booked = follow up
Phase 2 Backlog — Fix After Foundation Works 6

5. Orbit System Needs Full Build-Out

Plan mentions nurture side-tracks but the actual Orbit system has 5 reasons (No Response, Not Now, Ghost, Has Team, Hard No), two cadences (45d / 60d), and a resurrection flow. Add orbit_reason, orbit_cadence_days, orbit_touch_number, resurrect_count.

6. TG Stage Has Its Own Nudge Cadence

Path B leads in TG get nudged Day 2, 5, 10. Day 14 no booking → Orbit. New sequence type: tg_booking (Path B only).

7. Template Mapping Uses Old Stages

Section 4C maps templates to "Contacted" and "Qualified" — old stages. Re-map to new pipeline. Warm templates need to branch on dm_account (Path A → Calendly in DM; Path B → "drop your TG").

8. No Dedup Across Accounts

SDR DMs from multiple accounts — risk of same project contacted twice. Add dedup_key (project name + handle) and block duplicate creation.

9. Dashboard Doesn't Split by Account

"DMs Sent = 73" — from which account? Add dm_account as a dimension so the team can compare Path A vs Path B performance.

10. No-Show + Contract Alerts

Two gaps: (a) No-show — booked lead ghosts, 3-day reschedule window then Orbit. Add no_show flag. (b) Contract unsigned 3 days → alert Closer + Founder. Add contract_sent_date, contract_signed_date.

What the Plan Gets Right Keep
  • Bucket system (A/B/C) with auto-scoring — exactly matches the playbook
  • Sequence engine architecture (crm_sequences + steps + daily cron) — right approach, just needs more types
  • Dashboard layout and core metrics (SDR + Closer split, BAMFAM tracker, weekly TG report)

On the "70% Already Exists" Claim

Data infrastructure is at ~70% (templates, contacts, TG bot, stage history exist). But the playbook-driven workflow — two-path routing, bump sequences, Calendly tracking, Orbit — is new work. Phase 1 is bigger than the plan implies. Consider splitting into 1a (correct pipeline + core fields + cold sequence) and 1b (Calendly webhook + Orbit + dashboard splits).
